### 3. Static Event Names Implementation

**Problem**: Magic strings for HTMX events ("user-created", "product-updated")

**Solution**: Created `NetMX.Events` package with static event constants

**Implementation**:

**New Package**: `framework/NetMX.Events/`
```csharp
namespace NetMX.Events;

public static class DomainEvents
{
    public static class User
    {
        public const string Created = "user:created";
        public const string Updated = "user:updated";
        public const string Deleted = "user:deleted";
        // ...
    }
    
    public static class UI
    {
        public const string ShowToast = "ui:show-toast";
        public const string CloseModal = "ui:close-modal";
        // ...
    }
}
```

**CLI Integration**: Updated `GenerateCrudCommand`

**Generated Code - Controller**:
```csharp
using NetMX.Events;

[HttpPost]
public async Task<IActionResult> Create(CreateProductDto dto)
{
    var created = await _service.CreateAsync(dto);
    
    // Type-safe! IntelliSense support!
    this.HxTrigger(DomainEvents.Product.Created, new { id = created.Id });
    
    return await List();
}
```

**Generated Code - View**:
```html
@using NetMX.Events

<div hx-get="/Product/List" 
     hx-trigger="load, @DomainEvents.Product.Created from:body">
</div>
```

**Generated Code - Event Constants**:
```csharp
// Auto-generated: Events/DomainEvents.Product.cs
namespace NetMX.Events;

public static partial class DomainEvents
{
    public static class Product
    {
        public const string Created = "product:created";
        public const string Updated = "product:updated";
        public const string Deleted = "product:deleted";
    }
}
```

**Benefits**:
- ✅ IntelliSense support for event names
- ✅ Compile-time checking (no typos!)
- ✅ Refactoring safety ("Find All References")
- ✅ Self-documenting (XML docs describe payload)
- ✅ Extensible (partial classes for custom events)

## 💡 Key Decisions

### 1. Warning Suppression Strategy
**Decision**: Suppress via Directory.Build.props instead of fixing all 90 warnings

**Rationale**:
- Unblocks CI/CD immediately
- Industry standard approach
- Technical debt tracked
- Can add docs incrementally

**Trade-off**: Accept technical debt for speed

---

### 2. NuGet Publishing Strategy
**Decision**: Use NuGet.org with pre-release tags (not Azure Artifacts or MyGet)

**Rationale**:
- Simplest workflow (single feed)
- Zero additional cost
- Industry standard
- Planning open-source eventually

**Alternative Considered**: Azure Artifacts (private dev feed)  
**Why Rejected**: Additional infrastructure, overkill for current stage

---

### 3. Static Event Names Approach
**Decision**: Generate partial classes per entity instead of central registry

**Rationale**:
- Each entity owns its events
- Extensible via partial classes
- CLI can generate automatically
- No central bottleneck

**Alternative Considered**: Single DomainEvents class with all events  
**Why Rejected**: Would create merge conflicts, not scalable
