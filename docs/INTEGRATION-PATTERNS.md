## Overview

NetMX is a **modular monolith** framework. Features and modules need to communicate with each other in clean, maintainable ways.

### Two Main Integration Patterns

| Pattern | Scope | Use Case | Example |
|---------|-------|----------|---------|
| **HTMX Events** | UI (Browser) | One component reacts to another | Product created → Refresh inventory stats |
| **Service Injection** | Server (C#) | One feature needs another's logic/data | Order needs Product info |

Both patterns are **built into NetMX** and work out of the box with generated code.

---

## Pattern 1: HTMX Events (UI Integration)

### What It Is

HTMX events allow browser components to communicate **without knowing about each other**.

When something happens (product created, order placed), **trigger an event**. Any component can **listen** to that event and react.

### How It Works

**1. Controller triggers event**:
```csharp
[HttpPost]
public async Task<IActionResult> Create(CreateProductDto dto)
{
    await _service.CreateAsync(dto);
    
    // ✅ Trigger global HTMX event
    this.HxTrigger("product-created");
    
    return await List();
}
```

**2. Views listen to event**:
```html
<!-- Product list refreshes -->
<div id="product-list" 
     hx-get="/Product/List" 
     hx-trigger="load, product-created from:body">
    @* Product list here *@
</div>

<!-- Inventory stats also refresh (different component!) -->
<div id="inventory-stats" 
     hx-get="/Inventory/Stats" 
     hx-trigger="product-created from:body">
    <p>Total Products: <strong>@Model.TotalProducts</strong></p>
</div>

<!-- Recent activity updates too -->
<div id="recent-activity" 
     hx-get="/Activity/Recent" 
     hx-trigger="product-created from:body">
    @* Activity feed here *@
</div>
```

**Key Point**: `product-created` event triggers **3 different components** to refresh. They don't know about each other!

### Event Naming Convention

Use **feature-action** format:

```
product-created
product-updated
product-deleted
order-placed
order-shipped
user-registered
inventory-updated
```

### When to Use

✅ **Use HTMX events when**:
- One UI component needs to react to another
- Multiple components need to update on same action
- You want loose coupling between features
- Changes are visual (refresh list, update counts, etc.)

❌ **Don't use HTMX events when**:
- You need server-side business logic integration
- You need to validate/transform data between features
- You need transactional consistency

### Complete Example

**Scenario**: When Product is created, refresh product list AND inventory stats AND recent activity.

**ProductController.cs** (generated by CLI):
```csharp
[HttpPost]
public async Task<IActionResult> Create(CreateProductDto dto)
{
    if (!ModelState.IsValid)
        return PartialView("_Form", dto);

    await _service.CreateAsync(dto);
    
    // ✅ Trigger event - any component can listen
    this.HxTrigger("product-created");
    
    return await List();
}
```

**Product/Index.cshtml**:
```html
<div class="columns">
    <div class="column is-8">
        <!-- Product list -->
        <div id="product-list" 
             hx-get="/Product/List" 
             hx-trigger="load, product-created from:body"
             hx-swap="innerHTML">
            @await Html.PartialAsync("_List", Model)
        </div>
    </div>
    
    <div class="column is-4">
        <!-- Sidebar stats (different feature!) -->
        <div id="inventory-summary" 
             hx-get="/Inventory/Summary" 
             hx-trigger="load, product-created from:body"
             hx-swap="innerHTML">
            @* Inventory stats here *@
        </div>
    </div>
</div>
```

**Inventory/Summary.cshtml** (partial):
```html
<div class="box">
    <h3 class="title is-5">Inventory Summary</h3>
    <p>Total Products: <strong>@Model.TotalProducts</strong></p>
    <p>Low Stock: <strong>@Model.LowStockCount</strong></p>
    <p>Last Updated: <strong>@DateTime.Now.ToString("HH:mm")</strong></p>
</div>
```

**Result**: When product is created, **3 things happen automatically**:
1. ✅ Product list refreshes
2. ✅ Inventory stats update
3. ✅ All without page reload!

## Pattern 2: Service Injection (Direct Integration)

### What It Is

When one feature needs **data or logic** from another feature, inject the service directly.

This is standard **Dependency Injection** - the feature you need is injected into your service.

### How It Works

**1. Define interface** (auto-generated by CLI):
```csharp
public interface IProductService : IScopedDependency
{
    Task<ProductDto?> GetByIdAsync(Guid id);
    Task<List<ProductDto>> GetAllAsync();
    // ... other methods
}
```

**2. Inject into other service**:
```csharp
public class OrderService : IOrderService
{
    private readonly IOrderRepository _orderRepository;
    private readonly IProductService _productService;  // ✅ Inject Product service
    
    public OrderService(
        IOrderRepository orderRepository, 
        IProductService productService)
    {
        _orderRepository = orderRepository;
        _productService = productService;
    }
    
    public async Task<OrderDto> CreateOrderAsync(CreateOrderDto dto)
    {
        // ✅ Get product information
        var product = await _productService.GetByIdAsync(dto.ProductId);
        
        if (product == null)
            throw new BusinessException("Product not found");
        
        if (!product.IsActive)
            throw new BusinessException("Product is not available");
        
        // ✅ Use product data to create order
        var order = new Order(
            id: Guid.NewGuid(),
            productId: product.Id,
            productName: product.Name,
            quantity: dto.Quantity,
            unitPrice: product.Price
        );
        
        await _orderRepository.AddAsync(order);
        
        return MapToDto(order);
    }
}
```

**3. Automatic registration** (via marker interface):
```csharp
// ✅ IScopedDependency automatically registers service
public class ProductService : IProductService
{
    // Implementation...
}
```

### When to Use

✅ **Use service injection when**:
- One feature needs data from another feature
- You need business logic integration
- You need validation or transformation
- You need transactional consistency
- Both features are in the same module or tightly related

❌ **Don't use service injection when**:
- Features are in different bounded contexts (use events instead)
- You want loose coupling for future microservices
- You only need UI updates (use HTMX events)

### Complete Example

**Scenario**: Order service needs Product info when creating an order.

**Generated by CLI**:
```bash
netmx generate feature Product
netmx generate feature Order
```

**ProductService.cs** (generated):
```csharp
public interface IProductService : IScopedDependency
{
    Task<ProductDto?> GetByIdAsync(Guid id);
    Task<List<ProductDto>> GetAllAsync();
    Task<ProductDto> CreateAsync(CreateProductDto dto);
    Task UpdateAsync(UpdateProductDto dto);
    Task DeleteAsync(Guid id);
}

public class ProductService : IProductService
{
    private readonly AppDbContext _context;
    
    public ProductService(AppDbContext context)
    {
        _context = context;
    }
    
    public async Task<ProductDto?> GetByIdAsync(Guid id)
    {
        return await _context.Products
            .Where(p => p.Id == id)
            .Select(p => new ProductDto
            {
                Id = p.Id,
                Name = p.Name,
                Description = p.Description,
                Price = p.Price,
                IsActive = p.IsActive
            })
            .FirstOrDefaultAsync();
    }
    
    // ... other methods
}
```

**OrderService.cs** (add ProductService injection):
```csharp
public class OrderService : IOrderService
{
    private readonly AppDbContext _context;
    private readonly IProductService _productService;  // ✅ Add this
    
    public OrderService(
        AppDbContext context,
        IProductService productService)  // ✅ Inject here
    {
        _context = context;
        _productService = productService;
    }
    
    public async Task<OrderDto> CreateAsync(CreateOrderDto dto)
    {
        // ✅ Validate product exists and is active
        var product = await _productService.GetByIdAsync(dto.ProductId);
        
        if (product == null)
            throw new BusinessException("Product not found");
        
        if (!product.IsActive)
            throw new BusinessException("Product is not available");
        
        // ✅ Create order with product information
        var order = new Order
        {
            Id = Guid.NewGuid(),
            ProductId = product.Id,
            ProductName = product.Name,
            Quantity = dto.Quantity,
            UnitPrice = product.Price,
            TotalPrice = product.Price * dto.Quantity,
            CreatedAt = DateTime.UtcNow
        };
        
        _context.Orders.Add(order);
        await _context.SaveChangesAsync();
        
        return new OrderDto
        {
            Id = order.Id,
            ProductName = order.ProductName,
            Quantity = order.Quantity,
            TotalPrice = order.TotalPrice
        };
    }
}
```

**Result**: Order service can safely create orders with validated product information.

## Pattern 3: Shared DTOs

### What It Is

When multiple features need to work with the same data shape, use **shared DTOs**.

### Project Structure

```
MyApp.Web/
├── Dtos/
│   ├── Products/          # Product-specific DTOs
│   │   ├── ProductDto.cs
│   │   ├── CreateProductDto.cs
│   │   └── UpdateProductDto.cs
│   ├── Orders/            # Order-specific DTOs
│   │   ├── OrderDto.cs
│   │   ├── CreateOrderDto.cs
│   │   └── UpdateOrderDto.cs
│   └── Shared/            # ✅ Shared across features
│       ├── AddressDto.cs
│       ├── MoneyDto.cs
│       └── PagedResultDto.cs
```

### Example

**Shared/AddressDto.cs**:
```csharp
namespace MyApp.Dtos.Shared;

public class AddressDto
{
    public string Street { get; set; } = string.Empty;
    public string City { get; set; } = string.Empty;
    public string State { get; set; } = string.Empty;
    public string ZipCode { get; set; } = string.Empty;
    public string Country { get; set; } = string.Empty;
}
```

**Used by multiple features**:
```csharp
// Orders/CreateOrderDto.cs
public class CreateOrderDto
{
    public Guid ProductId { get; set; }
    public int Quantity { get; set; }
    public AddressDto ShippingAddress { get; set; } = new();  // ✅ Shared DTO
}

// Customers/CustomerDto.cs
public class CustomerDto
{
    public Guid Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public AddressDto BillingAddress { get; set; } = new();  // ✅ Same DTO
    public AddressDto ShippingAddress { get; set; } = new();
}
```

## Decision Guide

### When Feature A Needs Feature B

```
┌─────────────────────────────────────────────────────┐
│ What do you need?                                   │
└─────────────────────────────────────────────────────┘
             │
             ├─ Just UI update?
             │  → ✅ HTMX Events
             │     Controller: this.HxTrigger("event-name")
             │     View: hx-trigger="event-name from:body"
             │
             ├─ Need data or logic from another feature?
             │  → ✅ Service Injection
             │     Constructor: IProductService _productService
             │     Method: await _productService.GetByIdAsync(id)
             │
             └─ Need to share data structures?
                → ✅ Shared DTOs
                   Put in Dtos/Shared/ folder
```

### Integration Matrix

| Your Need | Pattern | Coupling | Future-Proof |
|-----------|---------|----------|--------------|
| UI component refresh | HTMX Events | ✅ Loose | ✅ Yes |
| Get data from another feature | Service Injection | ⚠️ Medium | ⚠️ Harder to split |
| Validate business rules | Service Injection | ⚠️ Medium | ⚠️ Harder to split |
| Share data structures | Shared DTOs | ✅ Loose | ✅ Yes |
| Trigger side effects | Domain Events* | ✅ Loose | ✅ Yes |

*Domain Events coming soon

## Real-World Examples

### Example 1: E-Commerce Product & Order

**Features**: Product, Order, Inventory

**Integration Needs**:
1. Order needs Product info (price, name)
2. Creating order should update inventory
3. UI should show updated inventory stats

**Solution**:
```csharp
// OrderService.cs
public class OrderService : IOrderService
{
    private readonly IProductService _productService;      // ✅ Service injection
    private readonly IInventoryService _inventoryService;  // ✅ Service injection
    
    public async Task<OrderDto> CreateAsync(CreateOrderDto dto)
    {
        // ✅ Get product info
        var product = await _productService.GetByIdAsync(dto.ProductId);
        
        // ✅ Check inventory
        var available = await _inventoryService.CheckStockAsync(dto.ProductId, dto.Quantity);
        if (!available)
            throw new BusinessException("Insufficient stock");
        
        // ✅ Create order
        var order = new Order(dto.ProductId, product.Name, product.Price, dto.Quantity);
        await _repository.AddAsync(order);
        
        // ✅ Update inventory
        await _inventoryService.ReduceStockAsync(dto.ProductId, dto.Quantity);
        
        return MapToDto(order);
    }
}
```

```csharp
// OrderController.cs
[HttpPost]
public async Task<IActionResult> Create(CreateOrderDto dto)
{
    await _service.CreateAsync(dto);
    
    // ✅ Trigger HTMX events for UI updates
    this.HxTrigger("order-created");
    this.HxTrigger("inventory-updated");  // Inventory stats refresh
    
    return await List();
}
```

```html
<!-- Multiple components react to events -->
<div id="order-list" 
     hx-get="/Order/List" 
     hx-trigger="order-created from:body">
</div>

<div id="inventory-stats" 
     hx-get="/Inventory/Stats" 
     hx-trigger="inventory-updated from:body">
</div>

<div id="low-stock-alert" 
     hx-get="/Inventory/LowStockAlert" 
     hx-trigger="inventory-updated from:body">
</div>
```

**Features**: BlogPost, Comment, ActivityFeed

**Integration Needs**:
1. Comment references BlogPost
2. Creating comment updates activity feed

public class CommentService : ICommentService

## Best Practices

### DO ✅

1. **Use HTMX events for UI updates**
   - Multiple components can react independently
   - Loose coupling between features
   - Works great for modular monolith

2. **Use service injection for business logic**
   - Type-safe, compile-time checked
   - Easy to test with mocks
   - Clear dependencies

3. **Name events consistently**
   - Format: `feature-action` (e.g., `product-created`)
   - Document available events
   - Trigger from controller after successful action

4. **Keep services focused**
   - One service per feature
   - Clear, single responsibility
   - Easy to understand and maintain

5. **Use shared DTOs for common data**
   - Put in `Dtos/Shared/` folder
   - Reuse across features
   - Reduces duplication

### DON'T ❌

1. **Don't create circular dependencies**
   ```csharp
   // ❌ BAD: OrderService → ProductService → OrderService
   public class ProductService
   {
       private readonly IOrderService _orderService;  // ❌ Creates cycle
   }
   ```

2. **Don't trigger events in loops**
   ```csharp
   // ❌ BAD: Trigger event for each item
   foreach (var product in products)
   {
       this.HxTrigger("product-updated");  // ❌ Too many events
   }
   
   // ✅ GOOD: Trigger once after batch
   // ... process all products ...
   this.HxTrigger("products-batch-updated");
   ```

3. **Don't use service injection across bounded contexts**
   ```csharp
   // ⚠️ QUESTIONABLE: Orders module directly calling Shipping module
   public class OrderService
   {
       private readonly IShippingService _shippingService;  // ⚠️ Tight coupling
   }
   
   // ✅ BETTER: Use domain events (coming soon)
   ```

4. **Don't put business logic in controllers**
   ```csharp
   // ❌ BAD: Business logic in controller
   [HttpPost]
   public async Task<IActionResult> Create(CreateOrderDto dto)
   {
       var product = await _productService.GetByIdAsync(dto.ProductId);
       if (product.Price > 1000) { /* logic */ }  // ❌ Should be in service
   }
   
   // ✅ GOOD: Logic in service
   [HttpPost]
   public async Task<IActionResult> Create(CreateOrderDto dto)
   {
       await _service.CreateAsync(dto);  // ✅ Service handles logic
       this.HxTrigger("order-created");
       return await List();
   }
   ```
