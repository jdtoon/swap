## 📡 Three Core Libraries

### 1. **Swap.Htmx** — HTMX Navigation Framework

HTMX-first MVC framework with automatic page/partial detection, event-driven UI, and middleware enforcement.

**What it does:**
- `SwapController` base class — Automatically detects HTMX vs. normal requests
- `SwapView()` helper — Returns `PartialView()` or `View()` based on request type
- Event bus with chain resolution — Declarative server events mapped to HTMX triggers
- Middleware enforcement — Catches full-page responses sent to HTMX requests
- Strongly-typed event keys — Roslyn analyzer flags magic strings (SWAPHTMX001)

**Install:**
```bash
dotnet add package Swap.Htmx
```

**Get Started:**
- [Swap.Htmx/README.md](./Swap.Htmx/README.md) — Full documentation and examples
- [35 tests](./Swap.Htmx.Tests/) — Test suite demonstrating all features

---

### 2. **Swap.Modularity** — Modular Monolith Toolkit

Discover, organize, and coordinate independent modules within a single deployable. Provides module discovery, endpoint mapping, and event chain orchestration.

**What it does:**
- IModule contract — Define modules with dependencies and initialization
- Discovery & ordering — Automatically discover and topologically sort modules
- Endpoint mapping — Register each module's controllers and endpoints
- MVC ApplicationParts — Load module UI assemblies (.Web projects) into the app
- Event chain integration — Coordinate events across module boundaries

**Install:**
```bash
dotnet add package Swap.Modularity
```

**Get Started:**
- [Swap.Modularity/README.md](./Swap.Modularity/README.md) — Full documentation and examples
- Use with modular monolith template: `swap new MyApp --template swap-modular-monolith`

---

### 3. **✅ Swap.Testing** — HTMX Test Framework

Fluent testing utilities for HTMX applications. Built on WebApplicationFactory with rich assertions for DOM, HTMX headers, and partial responses.

**What it does:**
- `HtmxTestClient` — Makes HTMX requests with proper `HX-Request` header
- CSS selector queries — Query HTML responses using CSS selectors
- HTMX-aware assertions — Assert on HX-Trigger, HX-Redirect, partial vs. full-page
- Snapshot testing — Compare HTML snapshots with scrubber support
- Fragment validation — Ensure partial views are returned for HTMX requests

**Install:**
```bash
dotnet add package Swap.Testing
```

**Get Started:**
- [Swap.Testing/README.md](./Swap.Testing/README.md) — Full documentation and examples
- Every generated project includes integration tests using this library

## 🎨 Bonus: Swap.Patterns — Entity Patterns Library

Common data modeling patterns ready to apply to your entities.

**What it provides:**
- 🗑️ **Soft Delete** — Logical deletion with restore capability
- 📝 **Auditable** — Track who created/modified/deleted
- 🔗 **Sluggable** — URL-friendly slugs for entities
- ⏰ **Timestampable** — Auto CreatedAt/UpdatedAt
- 📅 **Publishable** — Draft/published workflow with scheduling
- 🔢 **Orderable** — Display order and position management
- 🔖 **Versionable** — Version tracking
- 👁️ **Visibility** — Public/private with time-based scheduling

**Apply to your entities:**
```bash
swap generate pattern softdelete Post --use-package
```

**Install:**
```bash
dotnet add package Swap.Patterns
```

**Learn more:**
- [Swap.Patterns/README.md](./Swap.Patterns/README.md) — Pattern documentation
- [72 tests](./Swap.Patterns.Tests/) — Comprehensive test suite

## 🚀 Getting Started

### Using All Three in a Template

All Swap CLI templates ship with these libraries pre-configured:

```bash
# Create a modular monolith (uses Swap.Htmx + Swap.Modularity + Swap.Testing)
swap new MyApp --template swap-modular-monolith

# Or a layered app (uses Swap.Htmx + Swap.Testing)
swap new MyApp --template swap-layered
```

### Using in an Existing Project

Add individually or together:

```bash
# HTMX navigation framework
dotnet add package Swap.Htmx

# Module discovery and orchestration
dotnet add package Swap.Modularity

# Test utilities
dotnet add package Swap.Testing

# Entity patterns
dotnet add package Swap.Patterns
```

## 🔧 Local Development

### Build All Packages

```bash
cd framework
dotnet build
```

### Run All Tests

```bash
dotnet test
```

### Pack for Local Testing

Use the convenience scripts from the root:

```powershell
# Windows
.\scripts\pack-local.ps1

# Linux/Mac
./scripts/pack-local.sh
```

This builds and packs all framework packages to `.nuget/local/` for testing with `--local-nuget` flag.

### Test with Generated Projects

```bash
# Create a test project with local packages
swap new MyTestApp --local-nuget

# Test your framework changes
cd MyTestApp
dotnet restore
dotnet build
dotnet run
```

Each library solves one concern and can be used independently:

| Library | Use When |

| **Swap.Htmx** | Building any HTMX-powered ASP.NET Core app |

**Together:** The Swap ecosystem provides a complete, production-ready solution.
