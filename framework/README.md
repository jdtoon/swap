# Swap Framework Packages

This folder contains all NuGet packages that are used by projects generated by Swap CLI.

## Overview

The `framework/` directory includes both the packages and their test projects. These packages provide runtime functionality and testing utilities for applications generated with `swap new` and `swap generate` commands.

## Structure

```
framework/
├── Swap.Htmx/              # HTMX navigation framework
├── Swap.Htmx.Tests/        # Tests for Swap.Htmx (35 tests)
├── Swap.Patterns/          # Entity patterns library
├── Swap.Patterns.Tests/    # Tests for Swap.Patterns (72 tests)
├── Swap.Testing/           # HTMX testing framework
└── (Swap.Testing.Tests)    # (Not yet created)
```

## Packages

### 1. Swap.Htmx [![NuGet](https://img.shields.io/nuget/v/Swap.Htmx.svg)](https://www.nuget.org/packages/Swap.Htmx/)

**Purpose:** HTMX navigation framework for ASP.NET Core MVC.

**Features:**
- `SwapController` base class - Automatic page/partial detection
- `SwapView()` helper - Single method for page or partial rendering
- HTMX extension methods - Work with HX-* headers fluently
- Shell middleware - Enforce partial responses for HTMX requests

**Installation:**
```bash
dotnet add package Swap.Htmx
```

**Documentation:** [Swap.Htmx/README.md](./Swap.Htmx/README.md)

**Tests:** [Swap.Htmx.Tests/README.md](./Swap.Htmx.Tests/README.md) - 35 tests

### 2. Swap.Patterns [![NuGet](https://img.shields.io/nuget/v/Swap.Patterns.svg)](https://www.nuget.org/packages/Swap.Patterns/)

**Purpose:** Common entity patterns for data modeling.

**Patterns Included:**
- 🗑️ **Soft Delete** - Logical deletion with restore capability
- 📝 **Auditable** - Track who created/modified/deleted entities
- 🔗 **Sluggable** - URL-friendly slugs for entities
- ⏰ **Timestampable** - Automatic CreatedAt/UpdatedAt timestamps
- 📅 **Publishable** - Draft/published workflow with scheduling
- 🔢 **Orderable** - Display order and position management
- 🔖 **Versionable** - Version tracking for entities
- 👁️ **Visibility** - Public/private visibility with time-based scheduling

**Installation:**
```bash
dotnet add package Swap.Patterns
```

**CLI Integration:**
```bash
# Apply pattern to entity
swap g pattern softdelete Post --use-package
```

**Documentation:** [Swap.Patterns/README.md](./Swap.Patterns/README.md)

**Tests:** [Swap.Patterns.Tests/README.md](./Swap.Patterns.Tests/README.md) - 72 tests

### 3. Swap.Testing [![NuGet](https://img.shields.io/nuget/v/Swap.Testing.svg)](https://www.nuget.org/packages/Swap.Testing/)

**Purpose:** Fluent testing framework for HTMX applications.

**Features:**
- Fluent assertion API for HTMX responses
- `HtmxTestClient` with HX-Request header support
- HTML parsing and CSS selector queries
- Form submission helpers
- HX-Redirect following
- Snapshot testing with scrubbers
- Integration test base classes

**Installation:**
```bash
dotnet add package Swap.Testing
```

**CLI Integration:**
```bash
# Generate test class
swap g test Product
```

**Documentation:** [Swap.Testing/README.md](./Swap.Testing/README.md)

**Tests:** Not yet created (TODO for v0.2.0)

## Development Workflow

### Building Packages Locally

```bash
# Build all framework packages
cd framework
dotnet build

# Or build specific package
cd Swap.Htmx
dotnet build
```

### Running Tests

```bash
# Run all framework tests
cd framework
dotnet test

# Run specific test project
cd Swap.Patterns.Tests
dotnet test
```

### Packing for Local Testing

Use the root-level scripts:
```powershell
# Windows
.\scripts\pack-local.ps1

# Linux/Mac
./scripts/pack-local.sh
```

This builds and packs all framework packages to `.nuget/local/`.

### Testing with Generated Projects

```bash
# Generate project with local packages
swap new TestApp --local-nuget
cd TestApp
dotnet build
dotnet run
```

## CI/CD

Framework packages are automatically built, tested, and published via GitHub Actions:

**Build:** `.github/workflows/ci-build.yml`
- Builds all packages
- Runs all tests
- Packs NuGet packages
- Stores in `artifacts/packages/`

**Publish:** `.github/workflows/nuget-publish.yml`
- Publishes to NuGet.org on PR merge to `main`
- Creates GitHub release with packages
- Tags repository with version

## Adding New Packages

To add a new framework package:

1. **Create project:**
   ```bash
   cd framework
   dotnet new classlib -n Swap.NewPackage
   ```

2. **Add to solution:**
   ```bash
   cd ..
   dotnet sln add framework/Swap.NewPackage/Swap.NewPackage.csproj
   ```

3. **Configure `.csproj`:**
   ```xml
   <PropertyGroup>
     <TargetFramework>net9.0</TargetFramework>
     <Version>0.1.0</Version>
     <PackageId>Swap.NewPackage</PackageId>
     <Authors>Your Name</Authors>
     <Description>Package description</Description>
     <PackageTags>swap;aspnetcore;htmx</PackageTags>
     <RepositoryUrl>https://github.com/jdtoon/swap</RepositoryUrl>
     <PackageLicenseExpression>MIT</PackageLicenseExpression>
   </PropertyGroup>
   ```

4. **Add README.md** with usage examples

5. **Create test project:**
   ```bash
   dotnet new xunit -n Swap.NewPackage.Tests
   dotnet add Swap.NewPackage.Tests reference Swap.NewPackage
   dotnet sln add framework/Swap.NewPackage.Tests/Swap.NewPackage.Tests.csproj
   ```

6. **Update CI workflows** if needed
