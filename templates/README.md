# Templates

This folder contains all code generation templates used by Swap CLI.

## Overview

Templates are the heart of Swap CLI's code generation engine. Every file generated by `swap new` or `swap generate` commands originates from a template in this directory.

## Structure

```
templates/
├── monolith/          # Full project template (swap new)
├── swap-monolith/     # Monolith with Swap event system wired by default
├── swap-layered/      # Multi-project layered solution (Web, Application, Domain, Infrastructure)
└── generate/          # CRUD generation templates (swap generate)
    ├── auth/          # Authentication scaffolding
    ├── controller/    # CRUD controllers and views
    ├── factory/       # Test data factories
    ├── model/         # Entity models
    ├── seed/          # Database seeders
    └── test/          # Integration tests
```

## Template Categories

### 1. Monolith Template (`monolith/`)

**Used by:** `swap new <ProjectName>`

**Purpose:** Complete ASP.NET Core + HTMX project scaffold.

**Contains:**
- `Program.cs.template` - Application entry point with EF Core, middleware
- `Project.csproj.template` - NuGet dependencies, SDK version
- `appsettings.json.template` - Configuration with DB connection strings
- `docker-compose.yml.template` - Multi-container setup
- `Dockerfile.template` - Multi-stage build
- `package.json.template` - Tailwind CSS build setup
- `libman.json.template` - Client libraries (HTMX, DaisyUI)
- `tailwind.config.js.template` - Tailwind configuration
- `README.md.template` - Generated project documentation
- `Controllers/` - HomeController with HTMX examples
- `Views/` - Layout, home views, partials
- `Data/` - ApplicationDbContext
- `Models/` - Base model classes
- `Extensions/` - Helper extensions
- `wwwroot/` - Static assets

**Template Variables:**
- `{{ProjectName}}` - User-provided project name
- `{{DatabaseProvider}}` - sqlite, sqlserver, or postgres
- `{{UseLocalNuget}}` - true/false for local NuGet feed

### 2. Swap Monolith Template (`swap-monolith/`)

Used by: `swap new <ProjectName> --template swap-monolith`

Purpose: Same as Monolith, but with the Event System fully wired out-of-the-box.

Adds:
- `Events/SwapEventChains.cs` with sensible default chains (use `EventNames.*` constants like `EventNames.Ui.RefreshList`, `EventNames.Ui.ToastSuccess`)
- `Program.cs` registers both `UseSwapHtmxShell()` and `UseSwapHtmx()` and calls `AddSwapHtmx(opts => SwapEventChains.Configure(opts))`

Notes:
- Existing `monolith` remains unchanged for parity and minimalism; `swap-monolith` is the DX-forward option.

### 3. Layered Solution Template (`swap-layered/`)

Used by: `swap new <ProjectName> --template layered` (alias) or `--template swap-layered`

Purpose: Multi-project solution enforcing clean layering and HTMX-native event system.

Projects:
- `Web/` (Presentation): Controllers, views, HTMX integration, Swap event chains, model binders
- `Application/`: Use cases/services, abstractions, DTOs, event emission via IEventBus
- `Domain/`: Entities and domain events (single source of truth for event names)
- `Infrastructure/`: EF Core DbContext and repositories (provider-specific packages)

Highlights:
- HTMX-native ui.* events and server-side Swap event chains out of the box
- Session support and dev endpoints when ASPNETCORE_ENVIRONMENT=Development
- Pre-wired demos: Todos, Stats, Components, Dynamic, Bulk operations

Post-create steps (performed by CLI when not skipping setup):
- Run npm install + libman restore in `Web/`
- Build CSS in `Web/` (Tailwind + DaisyUI)
- Create and apply EF Core migration using `-p Infrastructure -s Web`

Template Variables:
- `{{ProjectName}}`, `{{ProjectNameLower}}`, `{{DatabaseProvider}}`, `{{UseLocalNuget}}`

Aliases:
- `--template layered` and `--template swap-layered` are equivalent and point to `templates/swap-layered`.

### 4. Generate Templates (`generate/`)


**Used by:** `swap generate <type>` commands

#### `auth/` - Authentication Scaffolding
**Command:** `swap g auth`

**Generates:**
- `ApplicationUser.cs` - Identity user model
- `AuthController.cs` - Login, register, logout, password reset
- `LoginViewModel.cs`, `RegisterViewModel.cs`, etc.
- Views: `Login.cshtml`, `Register.cshtml`, etc.
- `_LoginPartial.cshtml` - Navbar authentication UI

**Variables:**
- `{{ProjectName}}` - Namespace
- `{{Timestamp}}` - Generated file timestamp

#### `controller/` - CRUD Generation
**Command:** `swap g controller <Entity>`

**Generates:**
- `{Entity}Controller.cs` - Full CRUD with pagination, search, sort, filter
- `Index.cshtml` - List view with table, search, pagination
- `Create.cshtml` - Modal create form
- `Edit.cshtml` - Modal edit form
- `Delete.cshtml` - Modal delete confirmation
- `Details.cshtml` - Detail view
- `_Row.cshtml` - Table row partial for HTMX swaps
- `{Entity}IndexViewModel.cs` - Pagination, filters
- `{Entity}CreateViewModel.cs` - Create form
- `{Entity}EditViewModel.cs` - Edit form

**Variables:**
- `{{EntityName}}` - Entity name (singular)
- `{{EntityNamePlural}}` - Plural form
- `{{Fields}}` - Field definitions
- `{{ProjectName}}` - Namespace

#### `model/` - Entity Model
**Command:** `swap g model <Entity>`

**Generates:**
- `{Entity}.cs` - Entity class with properties

**Variables:**
- `{{EntityName}}`
- `{{Fields}}`
- `{{ProjectName}}`

#### `seed/` - Database Seeder
**Command:** `swap g seed <Entity>`

**Generates:**
- `{Entity}Seeder.cs` - Bogus-based fake data generator

**Variables:**
- `{{EntityName}}`
- `{{Fields}}`
- `{{ProjectName}}`

#### `factory/` - Test Factory
**Command:** `swap g factory <Entity>`

**Generates:**
- `{Entity}Factory.cs` - Test data factory for integration tests

**Variables:**
- `{{EntityName}}`
- `{{Fields}}`
- `{{ProjectName}}`

#### `test/` - Integration Test
**Command:** `swap g test <Controller>`

**Generates:**
- `{Entity}ControllerTests.cs` - HTMX test scaffold using Swap.Testing

**Variables:**
- `{{EntityName}}`
- `{{ProjectName}}`

## Template Engine

Templates are processed by `tools/Swap.CLI/Infrastructure/TemplateEngine.cs`.

### Variable Substitution
```csharp
// Replace {{VariableName}} with value
{{ProjectName}} → MyApp
{{EntityName}} → Product
{{EntityNamePlural}} → Products
```

### Conditional Blocks
```csharp
// Database-specific code
{{#if_sqlite}}
    options.UseSqlite(connectionString);
{{/if_sqlite}}

{{#if_sqlserver}}
    options.UseSqlServer(connectionString);
{{/if_sqlserver}}

{{#if_postgres}}
    options.UseNpgsql(connectionString);
{{/if_postgres}}

// Local NuGet mode
{{#if_local_nuget}}
    <add key="local" value="../../.nuget/local" />
{{/if_local_nuget}}
```

### Field Iteration
Controllers and views iterate over field definitions:
```csharp
foreach (var field in fields)
{
    // Generate form inputs, table columns, properties
}
```

## Template Syntax

Templates use **plain C#/Razor syntax** with variable placeholders:

```csharp
namespace {{ProjectName}}.Controllers;

public class {{EntityName}}Controller : SwapController
{
    // Generated code...
}
```

**No special template language** - just C# with `{{variables}}`.

## Adding New Templates

To add a new template:

1. **Create template file** in appropriate `generate/` subfolder
2. **Add variable placeholders** (`{{VariableName}}`)
3. **Update command** in `tools/Swap.CLI/Commands/`
4. **Register template** in command's `ExecuteAsync()` method
5. **Test generation** with `swap g <type> --dry-run`

Example:
```csharp
// In GenerateXCommand.cs
var templatePath = Path.Combine(templatesDir, "generate", "mytype", "Template.cs.template");
var content = File.ReadAllText(templatePath);
var processed = TemplateEngine.Process(content, variables);
```

## Template Variables Reference

### Common Variables
- `{{ProjectName}}` - Project/namespace name
- `{{EntityName}}` - Entity name (singular, PascalCase)
- `{{EntityNamePlural}}` - Plural form
- `{{EntityNameLower}}` - Lowercase entity name
- `{{Timestamp}}` - Generation timestamp
- `{{DatabaseProvider}}` - sqlite, sqlserver, postgres
- `{{UseLocalNuget}}` - true/false

### Field Variables (for entity generation)
- `{{PropertyName}}` - Property name
- `{{PropertyType}}` - C# type (string, int, DateTime, etc.)
- `{{IsNullable}}` - true/false
- `{{IsFilterable}}` - true/false
- `{{DisplayName}}` - Human-readable name

## Notes

- **Version control:** All templates are tracked in git
- **File extensions:** Use `.template` suffix (e.g., `Program.cs.template`)
- **Testing:** Use `--dry-run` flag to preview generated code
- **Validation:** Templates should generate valid, compilable code
- **Consistency:** Follow existing template patterns and naming conventions
